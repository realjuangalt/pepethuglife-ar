What visitors load (GitHub Pages)
---------------------------------
Only files under this folder should be published. In the repo: Settings → Pages →
Build from branch "main", folder "/frontend".

Site root (/) serves index.html — placeholder landing until the public launch.
The AR experience lives at /ar.html (same directory as config.js, ar-app.js, lib/, media/).

The live site loads: index.html, ar.html, config.js, ar-app.js, lib/*.js, and media/*.
There is no Python, no Flask, and no server-side AI stack.

The MindAR library (lib/mindar-image-aframe.prod.js) includes TensorFlow.js for
image tracking on the GPU. That runs entirely in the browser for marker detection.

The "backend/" folder at the repo root is legacy. It is not deployed when Pages
uses the "/frontend" folder.

Media model (two-phase)
------------------------
- Phase 1: small muted loop MP4s per marker (fast to load; autoplay when tracked).
- Phase 2: large full-song MP4s; loaded only after the user taps "Full song".

Paths and filenames are set in config.js under AR_CONFIG.markers (loop + full).

GIFs: convert to MP4 for loops (see ../scripts/convert-gif-loops.sh). The AR
plane uses <video>, not raw GIF.

targets.mind (multiple images)
------------------------------
Compile ALL reference images you need into ONE .mind file, in the same order as
targetIndex in config.js (0 = first image added in the compiler, etc.).

1. Use the official compiler: https://hiukim.github.io/mind-ar-js-doc/tools/compile
2. Save as frontend/media/targets.mind
3. Keep the same path in:
   - AR_CONFIG.marker.src in config.js
   - mindar-image="imageTargetSrc: ..." in index.html
4. maxTrack in index.html must be >= number of targets (this project uses 2).

Official docs: https://hiukim.github.io/mind-ar-js-doc/
